HAID visualizer — prototype samples

Three throwaway wireframes to refine the visualizer before we build it for real. All driven by real data from HAID's own sessions (last 60 days). Self-contained — they open in Chrome with no build step, the delivery model proposed for haid report.

1 · Window overview →

The zoom-out level: "how am I doing?" — waste placed vs baseline, work grouped into episodes. The navigation root (window › episode › session › diagram).

real metric headline · 16 sessions

2 · The bus diagram (flagship) →

The leaf view: time-layered spine, file buses in the gutters (left=reads, right=writes), width=token volume, right margin=cumulative cost. 3 themes, waste overlays toggle.

real session spine · 96–287 tool-calls

3 · Visual vocabulary →

Every channel in isolation — width tiers, file palette, edge glyphs, overlay badges, cost margin, the deferred token-type ribbon. Each panel ends with the open decision it teases out.

design system · 8 panels
Format proposal: a single self-contained HTML artifact written by haid report, opened in any browser — no server, no node_modules, matches the pip-install-and-go UX. Layout is hand-rolled SVG here (step-based Y, fixed lanes); ELK is the candidate if we need real crossing-minimization at scale.
